# Swift Mountain Improvements Plan

**Goal:** Elevate the CyberSlots demo site across many dimensions: performance, maintainability, UX, accessibility, SEO, code quality, and architecture. The project is a polished casino-themed React 19 + Vite + Tailwind site with an interactive multi-game slot machine (GSAP + Web Audio) and a separate MUI-based "Superpowers" feature.

**Current State Analysis (from exploration):**
- Strong foundations: Lazy loading with IntersectionObserver, responsive AVIF/WebP images, strict TS, custom casino theme, playable slot logic (4 games: neon, cascade, vault, egypt), promotions/VIP/leaderboard sections, reservation modal.
- Issues identified:
  - Legacy `casino/` directory (full old Convex + duplicate node_modules + configs) — ignored in ESLint but bloats repo, causes confusion.
  - Leftover `temp.tsx` (shader experiment) and `temp_utf8.tsx`.
  - UI inconsistency: Main site uses Tailwind + Radix/shadcn/ui + custom CSS; `/superpowers` uses MUI v9 + TanStack Query + Emotion. Theme mismatch.
  - `test-results/` folder with Playwright artifacts (should be .gitignore'd).
  - Slot machine is complex (1218 lines in PlayableSlotSection.tsx) — potential for optimization, better state mgmt, mobile touch, more polish.
  - SEO: Basic meta + generated sitemap/robots, but no JSON-LD structured data, no dynamic OG images, VITE_SITE_URL replacement may be fragile in build.
  - Build: Custom `build.mjs` runs tsc + vite; image opt and SEO scripts.
  - No analytics, limited error reporting beyond ErrorBoundary.
  - Playwright tests exist but browsers not installed in env; some layout assertions brittle.
  - Minor: Hardcoded player counts, no real persistence beyond localStorage for active slot, potential perf in heavy GSAP/animations on low-end devices.
  - Dependencies: Mix of modern (React 19) but heavy (MUI + full Radix suite + GSAP + recharts).

**Plan Structure:** Phased, prioritized improvements executable iteratively. Focus on high-impact, low-risk changes first. Total estimated effort: 2-4 dev days for core + testing.

## Phase 1: Cleanup & Housekeeping (Quick Wins, 1-2 hours)
1. Delete legacy `app/casino/` directory entirely (rm -rf safe after git status check). Update any references in docs/scripts if any. This removes ~1000+ files, duplicate deps, old Convex code.
2. Delete `app/temp.tsx`, `app/temp_utf8.tsx`, and any other temp/experiment files.
3. Add `casino/`, `test-results/`, `*.log`, `vite-*.log` to `.gitignore` (root + app if needed). Commit the cleanup.
4. Update `eslint.config.js` ignore if needed post-cleanup.
5. Run `npm run lint` and `npm run build` post-clean to validate.

**Deliverable:** Cleaner repo, faster clones, no confusion for future devs.

## Phase 2: Architecture & Consistency Unification (High Impact)
1. **UI Library Decision:** Standardize on Tailwind + shadcn/Radix (already dominant). 
   - Refactor `/superpowers` route and `features/superpowers/` to use existing casino theme, Tailwind classes, and shadcn components instead of MUI.
   - Remove MUI/Emotion deps if unused after (or keep for migration buffer).
   - Create a shared `CasinoThemeProvider` or extend Tailwind config for superpowers styles.
   - Migrate SuperpowerList/Card to match aesthetic (neon/casino-gold colors, serif/mono fonts).
2. **Feature Folder Expansion:** Move main Home sections (Hero, PlayableSlot, etc.) into `src/features/` or `src/sections/` consistently. Extract reusable hooks (e.g., useSlotMachine, useBonusState) from PlayableSlotSection.
3. **State Management:** For complex slot:
   - Introduce `useReducer` or lightweight store (no new deps: use existing Context + reducer) for bonus/spin state to reduce prop drilling and improve predictability.
   - Persist more state (VIP points, tickets) with localStorage + hydration.
4. **Routing & Pages:** Keep React Router; add error pages, 404. Consider React 19 `use` for data loading if applicable.
5. **Types & Utils:** Centralize types in `src/types/` or feature folders. Improve `SlotGameConfig`, `SpinResult` interfaces. Add Zod schemas for any future API (currently mock in superpowersApi).

**Tradeoff Note:** Full MUI removal saves ~50-100KB bundle but requires ~4-6h refactor. Alternative: Theming MUI to match casino (faster but inconsistent long-term).

## Phase 3: Performance & Optimization (Measurable Gains)
1. **Bundle & Build:**
   - Run `vite build --analyze` (or add rollup-plugin-visualizer temporarily) to identify heavy chunks (GSAP, recharts, MUI if kept).
   - Dynamic import more (e.g., Leaderboard only on scroll already lazy; split slot games configs).
   - Set `chunkSizeWarningLimit: 800` or optimize PlayableSlotSection by code-splitting per game.
   - Use Vite's `optimizeDeps` for lucide-react if icons bloat.
2. **Images & Assets:**
   - Enhance `scripts/optimize-images.mjs` (uses Sharp) to generate more sizes, add AVIF priority, WebP fallback automation.
   - Preload critical fonts/images already in index.html — verify in production build.
   - Add `loading="eager"` strategically, `decoding="async"`.
3. **Slot Machine Specific:**
   - Memoize symbol grids, payline calcs with `useMemo`.
   - Throttle spin animations, use `will-change` CSS.
   - Offload heavy GSAP ScrollTrigger to only when section visible.
   - AudioEngine: Lazy-load on first interaction, reduce oscillator count for ambient.
   - Mobile: Add touch/swipe support for reel "spinning" simulation.
4. **Runtime:**
   - Enable React Compiler? (experimental, test perf first).
   - Add `react-scan` or similar for dev profiling (dev dep).
   - Content-visibility and contain-intrinsic-size already used — extend to more sections.
   - Service worker? Optional for offline demo (PWA manifest).

**Expected:** 20-40% smaller initial bundle, faster TTI on mobile, smoother 60fps spins.

## Phase 4: UX, Accessibility & Polish
1. **Accessibility (WCAG 2.2 AA target):**
   - Audit with axe or Lighthouse: Add missing `aria-live` regions for spin results/wins (announce "Big win! +120 credits").
   - Keyboard: Full nav in slot (Tab to controls, Arrow keys for bet adjust, Enter/Space spin). Ensure all buttons have visible focus.
   - Color: Verify contrast ratios for gold/ember on dark ink (use tools).
   - Motion: Respect `prefers-reduced-motion` (already in scrollBehavior).
   - Modal: Improve ReservationModal focus trap, escape key (use existing Radix?).
   - Alt texts, labels already decent — enhance for featured games.
2. **Slot UX Enhancements:**
   - Add win celebrations (confetti already dep, trigger on big wins).
   - Better paytable modal with interactive highlights.
   - History log of last 5 spins.
   - Real-time leaderboard mock updates via interval (simulate live).
   - Mobile cabinet: Better hamburger in nav (tests cover), swipe gestures.
3. **General:**
   - Improve ReservationModal form with react-hook-form + Zod validation (already deps).
   - Add loading skeletons consistent with SectionPlaceholder.
   - Toast notifications (sonner already in deps) for bonus claims, wins.
   - Dark-only theme: Enforce with color-scheme.

## Implementation Order & Risks
- **Priority 1 (Immediate):** Phase 1 cleanup (removes bloat).
- **Priority 2:** Phase 2 unification + Phase 3 perf (biggest user impact).
- **Parallelizable:** SEO script updates, a11y audit can run alongside.
- **Risks:** MUI migration may introduce visual bugs — test thoroughly on mobile. Slot logic changes require regression via new tests. Large delete: backup branch.
- **Verification:** After each phase: `npm run lint && npm run build && npm run preview` + manual play + Playwright.
- **Tools:** Use existing (Vitest? add, Playwright, Lighthouse, axe).
