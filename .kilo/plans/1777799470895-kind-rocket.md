## 2. Core Mathematical Model
Define per-spin random variable X (net player result):
X = (winAmount * multiplier) - betAmount

House edge HE = -E[X] / betAmount > 0 (target 0.035–0.05)

RTP = 1 - HE (configurable base 0.96)

To enforce negative EV:
- Use weighted symbol distribution where Σ (p_i * payout_i) < 1.0
- Add global "loss bias" multiplier β (0.92–0.98) applied to high-payout weights when conditions met.
- Maintain a "virtual pool" (Redis sorted set + Postgres Transaction aggregate) tracking total wagered vs paid out; when pool ratio (paid/wagered) > 0.92, β decreases further.

Dynamic RTP formula (in GamesService):
```
baseRTP = 0.96
balanceFactor = clamp(1 - (currentBalance - initialBalance) / (initialBalance * 2), 0.85, 1.0)
poolFactor = clamp(1 - (poolPayoutRatio - 0.85) * 0.8, 0.88, 1.0)
winStreakFactor = clamp(1 - (recentWins / 5) * 0.05, 0.90, 1.0)
effectiveRTP = baseRTP * balanceFactor * poolFactor * winStreakFactor
```
Then scale symbol weights by (effectiveRTP / theoreticalRTP) before sampling.

This ensures that after a player reaches +50% balance or pool payout ratio >92%, win probability drops measurably.

## 4. Near-Miss Effect (Cognitive Bias Engine)
Goal: generate grids that visually appear one symbol away from jackpot/line win ≥12% of losing spins, without actually awarding payout.

Algorithm (in BiasedRNG):
- First sample a "target win symbol" using normal weights.
- With probability 0.12 (configurable nearMissRate):
  - Force two reels to the target symbol.
  - Sample third reel from "near-miss" distribution (exclude exact match, bias toward adjacent symbols in reel strip).
  - Set actual winAmount = 0 (or minimal consolation if configured).
- For hold-and-win games (vault/egypt): bias cash/lock symbols to appear in 2/3 positions but fail the 6-lock threshold.
- Visual cue in frontend: add CSS/GSAP "tease" animation on near-miss cells (pulse, highlight) + audio "almost" sting. Store `nearMiss: true` flag in SpinResult for UI.

Frontend update: PlayableSlotSection.tsx evaluate* functions must accept server-provided grid + nearMiss flag; never regenerate grid client-side.

## 6. Mathematical Justification & Long-Run Proof
House edge derivation:
Let p_w be probability of win category w with multiplier m_w.
E[X/bet] = Σ p_w * (m_w - 1) - (1 - Σ p_w) = (Σ p_w * m_w) - 1

With current backend multipliers and uniform reels: theoretical RTP ≈ 102% (positive EV, bug).
New weighted model targets Σ p*m = 0.96.

Proof of sustainability:
- By law of large numbers, after N spins, total player loss → N * |E[X]|
- Rare jackpots (0.08%) are funded by the 3.5% edge collected on every spin (including from other players).
- Dynamic β ensures that even if one player hits a 1000x jackpot, subsequent spins for that user have RTP ≤ 88% until balance normalizes.
- Redis pool + Postgres audit trail guarantees that aggregate payout ratio never sustainably exceeds 96.5%.
