# Plan: Optimize Website Layout and Cross-Device Testing for CyberSlots

## Overview
The goal is to audit, optimize the responsive layout of the CyberSlots site (React/Vite/Tailwind/GSAP/MUI), and verify it across devices and scenarios using code inspection + Playwright e2e device emulation. The site features a complex interactive slot machine, multi-section homepage, and MUI-based superpowers page. Current design uses mobile-first Tailwind (md:768px, lg:1024px, xl:1280px), fluid clamps, srcset images, overflow-x-hidden, and lazy loading - mostly solid but slot cabinet and some controls need mobile refinement.

## 1. Audit Current Layout (Read-Only Review)
- Inspect key files for responsive gaps:
  - `app/src/pages/Home.tsx`: grids (1/2/3, 1/2/4), px-[6vw], clamps, hero ticker.
  - `app/src/sections/PlayableSlotSection.tsx`: cabinet `max-w-[760px]`, symbol grid `gridTemplateColumns` + `aspect-square`, icons `h-9 md:h-11`, controls `flex-col lg:flex-row`, paytable `grid-cols-2 md:3`.
  - `app/src/sections/LeaderboardSection.tsx`, `FooterSection.tsx`, `Navigation.tsx`: lg: grids, md:flex.
  - `app/src/pages/Superpowers.tsx` + `features/superpowers/components/SuperpowerList.tsx` + `SuperpowerCard.tsx`: MUI Grid xs:12/md:6, Container pt:20.
  - `app/src/components/ReservationModal.tsx`: Dialog sm:max-w-md.
  - `app/src/index.css`, `tailwind.config.js`: custom casino theme, animations, no conflicting fixed sizes.
  - `app/index.html`: viewport meta present (good).
- Check for issues: small cells on 5x5 cascade game (<375px), touch targets, text overflow, horizontal scroll risk, inconsistent padding on superpowers.
- Review `use-mobile.ts` (768px breakpoint), lazy sections, GSAP triggers.
- Run `npm run lint` (app/) to baseline.

## 2. Optimization Tasks
- **Slot Machine (PlayableSlotSection.tsx: ~1000-1210)**: 
  - Add responsive icon sizing: `h-8 w-8 sm:h-9 md:h-11`.
  - Make cabinet more fluid: change to `w-full max-w-[min(100%,760px)]` or use `clamp`, adjust `min-h-[280px] sm:min-h-[300px]`.
  - Ensure controls stack cleanly on mobile (already lg:flex-row), add `flex-wrap` if needed.
  - Increase gap on small screens if cells crowd.
  - Verify spin button, bet input, stats grid (1/3/4) scale.
- **General Polish**:
  - Ensure all buttons/links meet 44px min touch target (Tailwind py-3+).
  - Add `touch-manipulation` or prevent zoom issues if any.
  - Superpowers: responsive Container padding (use sx responsive or Tailwind wrapper).
  - Footer/Nav: verify mobile menu and links.
  - Add any missing `sm:` / `lg:` variants for consistency.
  - Preserve performance: keep content-visibility, lazy, GSAP.
- After edits, run `npm run build` to confirm no breakage.
- No new deps; use existing Tailwind/MUI.

## 3. Device Testing Strategy (Playwright)
- @playwright/test is in devDeps. If no `playwright.config.ts`, create minimal one with device projects.
- Target devices/scenarios (use built-in `devices`):
  - **Mobile Portrait**: iPhone 12 (390px), Pixel 5 (393px), iPhone SE (375px).
  - **Mobile Landscape**: iPhone 12 landscape.
  - **Tablet**: iPad Mini (768px), iPad Pro (1024px).
  - **Desktop**: Desktop Chrome (1280px+), 1920px wide.
  - **Edge cases**: 320px narrow, 1440px, reduced-motion (prefers-reduced-motion), high-DPI (retina).
- **Test Scenarios** (create or extend tests in e.g. `app/tests/layout.spec.ts`):
  1. Load Home: no horizontal scroll/overflow, hero images load (srcset), nav fixed + mobile hamburger works, scroll to sections (games, play, jackpots, vip).
  2. Interactive Slot: select game (grid responsive), spin/auto-spin/reset, paytable opens/closes, balance/win updates visible, ticker, all on each viewport.
  3. Modal: open ReservationModal, form fields, validation, submit success on mobile/desktop.
  4. Superpowers page: /superpowers loads, MUI Grid 12/6 cols, cards stack/equip buttons usable on touch, GSAP animations.
  5. Navigation: cross-page links, scroll behavior.
  6. Footer: links, legal on mobile.
  7. Responsive images, text readability, no clipping.
- Run commands:
  - `npx playwright test --project="Mobile Chrome" --headed` (or specific projects).
  - Use `test.describe` per device or `test.use({ viewport: {width, height} })`.
  - Capture screenshots on failure + video traces.
  - Test both dev (`npm run dev`) and preview build.
- Verify: no console errors, smooth interactions, correct layouts.

## 4. Validation & Iteration
- After optimizations + tests pass: re-audit code, run full `npm run lint && npm run build`.
- If Playwright reports issues (e.g. small tap areas, overflow), fix iteratively.
- Check accessibility basics (labels, contrast - already dark theme).
- Optional: add to CI but keep scope to local verification.

## 5. Deliverables
- Optimized, responsive codebase.
- Playwright test suite covering devices/scenarios.
- Screenshots/video evidence of tests (if run).
- Confirmed no layout regressions across scenarios.

## Timeline / Order
1. Audit (quick).
2. Fix slot + minor responsive tweaks (edit PlayableSlotSection.tsx primarily).
3. Add/run device tests.
4. Final lint/build/verify.

This ensures production-ready layout optimization without over-engineering. Focus on mobile-first since interactive core (slot) is critical on phones/tablets.

## Notes
- Project root for commands: `app/`
- No secrets or external URLs.
- Existing lazy-loading, reduced-motion, and image optimization already help performance on devices.
- Tradeoff: Keep MUI in superpowers (consistent with current) vs full Tailwind migration (out of scope unless major issue found).
